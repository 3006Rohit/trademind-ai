🏗️ System Architecture

Complete overview of TradeMind AI's component architecture, including frontend components, services layer, data sources, and ML models ensemble.

Architecture Components

Frontend Layer

React components including TradingChart for visualization, MLPanel for predictions, OrderPanel for trading, and authentication interfaces.

Context & State

AuthContext manages global user sessions and application state, providing data to all components through React Context API.

Services Layer

Modular services for streaming, data processing, AI integration, and authentication providing clean separation of concerns.

Data Sources

Binance WebSocket for crypto, simulated data for stocks/forex, and Google Gemini API for AI-powered sentiment analysis.

Portfolio State

Centralized management of balance, open positions, closed positions, P&L tracking, and complete trade history per user.

ML Ensemble

6 models (LSTM, XGBoost, Random Forest, RNN, AdaBoost, ARIMA) working together to generate Buy/Sell/Hold signals.

🔄 Trading Workflow

Complete end-to-end workflow showing order execution, position management, ML predictions, and P&L calculations running in parallel.

Workflow Breakdown

1️⃣ Order Creation

User views prices in TradingChart and creates order via OrderPanel. Choose between Market (immediate) or Limit (conditional) order types.

2️⃣ Order Execution

Market orders execute immediately at current market price. Limit orders wait for price to match user's target or user cancels.

3️⃣ Position Entry

Position is created with entry price set to execution price. Balance is deducted. Position appears in portfolio tracking view.

4️⃣ Live Monitoring

Portfolio updates in real-time as market prices change. PnL updates continuously. User can monitor and decide when to close.

5️⃣ Position Closure

User closes position at current market price. Exit price is recorded. P&L calculated: Quantity x (Exit Price - Entry Price).

6️⃣ Settlement

Balance updated with realized P&L (profit or loss). Position moved to trade history. All changes persisted to localStorage.

ML Prediction Workflow

Parallel to order execution, ML models continuously analyze market data and generate trading recommendations:

Data Input

OHLCV data flows in real-time from Binance WebSocket or simulated data generators.

Feature Extraction

dataService processes candles and extracts features for model input.

Model Predictions

6 models run in parallel and predict next candle close price.

Signal Generation

Predictions converted to trading signals: STRONG BUY, BUY, HOLD, SELL, STRONG SELL.

Display

All signals displayed in MLPanel for user review and decision-making.

User Action

User accepts recommendation or ignores it, maintaining full control over trading decisions.
