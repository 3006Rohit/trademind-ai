🏗️ System Architecture

Complete overview of TradeMind AI's component architecture, including frontend components, services layer, and data sources.

Architecture Breakdown

Frontend Layer

React components for UI/UX including TradingChart, MLPanel, OrderPanel, and user authentication screens.

Context & State

AuthContext manages user sessions and global state, providing data to all components through React Context API.

Services Layer

Modular services for streaming, data processing, AI integration, and authentication - clean separation of concerns.

Data Sources

Real-time Binance WebSocket for crypto, simulated data for stocks/forex, and Google Gemini API for AI sentiment.

Portfolio State

Centralized portfolio management including balance, open positions, P&L tracking, and trade history.

ML Models

6 ensemble models (LSTM, XGBoost, Random Forest, RNN, AdaBoost, ARIMA) generate Buy/Sell/Hold signals.

🔄 Trading Workflow

Complete end-to-end workflow showing order execution, position management, ML predictions, and P&L calculations.

Workflow Steps

1️⃣ Order Creation

User views prices in TradingChart and creates order via OrderPanel (Market or Limit type).

2️⃣ Order Execution

Market orders execute immediately at current price. Limit orders await price match or user cancellation.

3️⃣ Position Entry

Position created with entry price. Balance deducted. Position appears in portfolio tracking view.

4️⃣ Real-time Monitoring

Portfolio updates in real-time as prices change. User can hold position or close it anytime.

5️⃣ Position Closure

User closes position at current price (exit price). P&L calculated: PnL = Quantity × Price Difference.

6️⃣ Settlement

Balance updated with P&L. Position moved to trade history. All data persisted to localStorage.

ML Prediction Parallel Workflow

While trading happens, ML models simultaneously analyze OHLCV data and generate buy/sell recommendations in MLPanel.

Data Flow Path

User Input → Components → Services → External APIs/Storage → State Update → UI Render

1. User initiates action (order creation, sentiment analysis)
2. Component sends request to appropriate service
3. Service processes data and calls external APIs if needed
4. Results stored in AuthContext/localStorage
5. Components re-render with updated data via React hooks
6. User sees real-time updates in UI
